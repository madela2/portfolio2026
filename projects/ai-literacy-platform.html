AI Literacy Platform (Full-stack MERN Application)

The AI Literacy Platform is a full-stack web application designed to help users recognize AI-generated content through interactive quizzes. The system supports Visitors, Researchers, and Admins with role-based access and secure authentication.

  • Team project
  • Course: Full-Stack Web Development
  • My role: Full-stack developer (authentication, quiz logic, dashboards)
  • Tech: React, Node.js, Express, MongoDB, JWT

Project overview

The AI Literacy Platform is an educational quiz system where users attempt to distinguish between AI-generated and human-created content, including images, audio, and video. The application collects quiz attempts and demographic data while allowing researchers to create and manage quizzes through a dashboard.

The platform includes secure authentication, email verification, role-based access control, analytics dashboards, and protected routing on both frontend and backend.

My role and contributions

I contributed to both backend and frontend development, with a strong focus on authentication, session management, and quiz functionality.

  • Implemented JWT authentication using HttpOnly cookies
  • Refactored token handling from localStorage to secure cookie-based sessions
  • Built email verification with 15-minute expiring token

  • Implemented inactivity timer with warning and automatic logout

  • Developed protected routes with loading and unauthorized handling
  • Contributed to researcher dashboard and admin analytics features

How it works

  1. Users register and verify their email through a time-limited verification link.
  2. Authenticated users receive role-based access (visitor, researcher, admin)
  3. Researchers create quizzes and upload media files.
  4. Visitors complete quizzes and submit demographic information.
  5. Quiz attempts are stored in MongoDB and processed for analytics.
  6. Dashboards display participation statistics and performance insights.

Challenges and solutions

  • Authentication refactor
    • Refactoring from localStorage to HttpOnly cookies required adjusting token handling, CORS configuration, protected routes, and session validation.
  • Media handling and file uploads
    • We encountered issues with video files not appearing during quizzes due to incorrect media paths in the seeding script. Debugging required inspecting backend routes, database entries, and static file configuration.
  • Quiz creation and publish logic
    • Draft updates and publish functionality required careful synchronization between frontend state management and backend PATCH routes.
  • Session and inactivity handling
    • Implementing inactivity timers while maintaining correct session persistence required restructuring AuthProvider logic and route protection.
