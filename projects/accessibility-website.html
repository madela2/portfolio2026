Accessibility-Focused Website

This project focused on designing and developing a small website with accessibility and usability in mind. The goal was to create a solution that supports keyboard navigation, semantic structure, and a clearer experience for a wide range of users.

  • Individual project
  • Course: Web Accessibility, Usability and Ethics
  • My role: Frontend developer
  • Tech: HTML, CSS
GitHub repository

Project overview

The purpose of this project was to build a small website that follows accessibility principles and supports a more user-friendly experience. I focused on semantic HTML, clear navigation, visible focus states, structured content, and accessible form design.

Even though this was a smaller frontend project, it was important to me because it allowed me to focus more deliberately on accessibility and usability. It helped me better understand how development choices can affect how easy a website is to use.

My role and contributions

I was responsible for the full implementation of the project, including structure, layout, and accessibility-related decisions.

  • Planned the page structure and navigation
  • Built the website using semantic HTML
  • Created a skip link for quicker keyboard navigation
  • Styled visible focus states for interactive elements
  • Built an accessible contact page with a contact table and form
  • Worked with labels, validation, and error messaging in the form

How it works

  1. Users can skip directly to the main content by using a skip link.
  2. The website uses semantic elements such as header, navigation, and main.
  3. The navigation is structured to be clear and easy to move through with a keyboard.
  4. The contact page includes contact information in a structured table.
  5. The contact form uses labels, required fields, and visual feedback to support usability.

Key features

  • Skip to main content link
  • Semantic page structure
  • Keyboard-friendly navigation
  • Visible focus styling
  • Accessible contact page
  • Contact form with labels and validation feedback

Challenges and solutions

  • Designing with accessibility in mind required me to think more carefully about structure, navigation, and focus visibility so the website would be easier to use.
  • Making the website work well without relying on a mouse required attention to keyboard navigation, tab order, and focus states.
  • Designing an accessible contact form meant working carefully with labels, required fields, and validation feedback to make the form easier to understand.

What I learned

  • How semantic HTML supports accessibility
  • Why visible focus states matter for keyboard users
  • How form structure and labels affect usability
  • How accessibility improvements can benefit all users, not only specific groups

Possible future improvements

  • Improve the responsive design for smaller screens
  • Refine the visual design further
  • Test with more accessibility tools and real users
  • Expand the website with more realistic content and pages
