Pawpartner - Dog Activity Tracker (Web of Things project)

Pawpartner is a Web of Things prototype designed to improve dogs' wellbeing and strengthen the dog-owner relationship. The system uses a wearable Micro:Bit sensor to track activity and presents the data in a web-based interface with actionable insights for new dog owners.

  • Team project with four other students
  • Course: Web of Things
  • My role: Frontend developer
  • Tech: Micro:Bit, Web Bluetooth, JavaScript, MongoDB

Project overview

Pawpartner is an activity-tracking system for dogs that combines hardware sensors and a web interface. A wearable MicroBit measures data, which is transmitted via Web Bluetooth to a web application. The data is stored in a database and visualozed to help owners understand whether their dog is getting enough daily activity. The project focuses on making activity data easy to understand and actionable, especially for new dog owners who may struggle to know how much exercise their dog needs.

Frontend developer, with main responsibility for the monthly activity overview.

  • Fetching and rendering activity data from backend APIs
  • Collaborating cloesly with backend and hardware contributors to integrate data flow
  • Participating in UI and usability decisions

How it works

  1. A Micro:Bit with a built-in accelerometer tracks the dog's movement.
  2. Activity data is sent to a web application via Web Bluetooth (BLE).
  3. The web application sends the data to a backend using REST APIs.
  4. Activity data is stored in a MongoDB database.
  5. The web interface requests stored data through APIs and displays activity summaries.

Technology stack

Hardware

  • Micro:Bit with built in accelerometer, Bluetooth and a battery bow with AAA-batteries

Frontend

  • JavaScript
  • HTML
  • CSS

Backend

  • REST APIs
  • MongoDB database

Why these choices?

  • The Micro:Bit provided all required sensors and Bluetooth out of the box
  • Accelerometers are commonly used in fitness trackers and step counters
  • JavaScript was sufficient due to the project's limited scope and kept complexity manageable
  • MongoDB suited the time-based activity data well

Key features

  • Wearable activity tracker for dogs
  • Adjustable sensitivity based on dog weight
  • Monthly activity overview
  • Activity history stored in a database
  • Personalized activity suggestions based on remaining daily activity needs
  • Relationship status calculated from the daily activity

Challenges and solutions

  • Bluetooth connectivity
    • Bluetooth connections are unstable and limited in range. To handle disconnections, the Micro:Bit stores activity data locally when offline and syncs it once the connection is restored.
  • Measuring activity accurately
    • We implemented movement and activity thresholds that adapt based on the dog's weight.
  • Data flow
    • Data is exchanged using UART over Web Bluetooth, parsed in the browser, and forwarded to the backend via REST APIs.
